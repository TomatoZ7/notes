# 工厂模式

万物皆对象，对象都需要创建，如果创建的时候直接 `new` 对象，就会对该对象耦合严重，加入我们要更换对象，所有 `new` 对象的地方都需要修改一遍，这显然违背了软件设计的开闭原则。如果我们使用工厂来生产对象，我们就只和工厂打交道就可以了，彻底和对象解耦，如果要更换对象，直接在工厂里更换该对象即可，达到了与对象解耦的目的。所以，工厂模式最大的优点就是：**解耦**。

本文主要介绍四种工厂模式：

+ 静态工厂模式（不属于 GOF 的 23 种经典设计模式）

+ 简单工厂模式（不属于 GOF 的 23 种经典设计模式）

+ 工厂方法模式

+ 抽象工厂模式

## 1.静态工厂模式（Static Factory）

此模式用于创建一系列相关或相互依赖的对象。静态工厂模式通常只使用一个静态方法来创建所有类型对象，此方法通常被命名为 `factory` 或 `build`。

### 1.3 实现

StaticFactory.php

```php
namespace DesignPatterns\Creational\StaticFactory;

/**
 * 1、静态意味着不能全局状态，因为它不能被模拟进行测试，所以它是有弊端
 * 2、不能被分类或模拟或多个不同的示例
 */
final class StaticFactory
{
    /**
     * @param string $type
     * 
     * @return FormatterInterface
     */
    public static function factory(String $type)
    {
        if ($type == 'number') {
            return new FormatNumber();
        }

        if ($type == 'string') {
            return new FormatString();
        }

        throw new \Exception('Unknown format given');
    }
}
```

FormatterInterface.php

```php

interface FormatterInterface
{

}
```

FormatString.php

```php
namespace DesignPatterns\Creational\StaticFactory;

class FormatString implements FormatterInterface
{
    
}
```

FormatNumber.php

```php
namespace DesignPatterns\Creational\StaticFactory;

{
    
}
```

### 1.4 测试

Tests/StaticFactoryTest.php

```php
namespace DesignPatterns\Creational\StaticFactory\Tests;

use DesignPatterns\Creational\StaticFactory\StaticFactory;
use PHPUnit\Framework\TestCase;

class StaticFactory extends TestCase
{
    public function testCanCreateNumberFormatter()
    {
        $this->assertInstanceOf(
            'DesignPatterns\Creational\StaticFactory\FormatNumber',

        )

    public function testCanCreateStringFormatter()
    {

            StaticFactory::factory('string')

    }

    public function testException()
    {
        StaticFactory::factory('object');
    }
}
```

## 2.简单工厂模式（Simple Factory）

简单工厂模式与静态工厂模式最大的区别是它不是**静态**的。因为非静态，所以你可以拥有多个不同参数的工厂。你可以为其创建子类，甚至可以 mock 他。

### 2.1 结构

简单工厂模式包含如下角色：

+ 抽象产品：定义了产品的规范，描述了产品的主要特性和功能；

+ 具体产品：实现或继承抽象产品的子类

+ 具体工厂：提供了创建产品的方法，调用者通过该方法来获取产品。

### 2.2 UML 类图

![image:factory_1](https://github.com/TomatoZ7/notes-of-tz/blob/master/Programming/DesignPatterns/images/factory_1.jpg)

### 2.3 实现

SimpleFactory.php

```php
namespace DesignPatterns\Creational\SimpleFactory;

class SimpleFactory 
{
    public function createBicycle(): Bicycle
    {
        return new Bicycle();
    }
}
```

Bicycle.php

```php
namespace DesignPatterns\Creational\Bicycle;

class Bicycle
{
    public function driveTo(string $destination)
    {
        return $destination;
    }
}
```

### 2.4 测试

```php
$factory = new SimpleFactory();
$bicycle = $factory->createBicycle();
$bicycle->driveTo('Paris');
```

## 3.工厂方法模式（Factory Method）

对比简单工厂模式的优点是，你可以将子类用不同的方法来创建一个对象。

举一个简单的例子，这个抽象类可能只是一个接口。

这种模式是「真正」的设计模式，因为它实现了 S.O.L.I.D 原则中 D 的「依赖倒置」。

这意味着工厂方法模式取决于抽象类，而不是具体的类。

### 3.1 UML 类图

![image:factory_3](https://github.com/TomatoZ7/notes-of-tz/blob/master/Programming/DesignPatterns/images/factory_3.jpg)

### 3.2 实现

Logger.php

```php
namespace DesignPattern\Creational\FactoryMethod;

interface Logger
{
    public function log(string $message);
}
```

StdoutLogger.php

```php
namespace DesignPattern\Creational\FactoryMethod;

class StdoutLogger implements Logger
{
    public function log(string $message)
    {
        echo $message;
    }
}
```

FileLogger.php

```php
namespace DesignPattern\Creational\FactoryMethod;

class FileLogger implements Logger
{
    private $filePath;

    public function __construct(string $filePath) {
        this->$filePath = $filePath;
    }

    public function log(string $message)
    {
        file_put_contents($this->filePath, $message . PHP_EOL, FILE_APPEND);
    }
}
```

LoggerFactory.php

```php
namespace DesignPattern\Creational\FactoryMethod;

interface LoggerFactory
{
    public function createLogger(): Logger;
}
```

StdoutLoggerFactory

```php
namespace DesignPattern\Creational\FactoryMethod;

class StdoutLoggerFactory
{
    public function createLogger()
    {
        return new StdourLogger();
    }
}
```

FileLoggerFactory.php

```php
namespace DesignPattern\Creational\FactoryMethod;

class FileLoggerFactory
{
    private $filePath;

    public function __construct(string $filePath)
    {
        this->$filePath = $filePath;
    }

    public function createLogger()
    {
        return new FileLogger($this->filePath);
    }
}
```

### 3.3 测试

```php
namespace DesignPattern\Creational\FactoryMethod\Tests;

use DesignPatterns\Creational\FactoryMethod\FileLogger;
use DesignPatterns\Creational\FactoryMethod\FileLoggerFactory;
use DesignPatterns\Creational\FactoryMethod\StdoutLogger;
use DesignPatterns\Creational\FactoryMethod\StdoutLoggerFactory;
use PHPUnit\Framework\TestCase;

class FactoryMethodTest extends TestCase
{
    public function testCanCreateStdoutLogger()
    {
        $loggerFactory = new StdoutLoggerFactory();
        $logger = $loggerFactory->createLogger();
        $this->assertInstanceOf(StdoutLogger::class, $logger);
    }

    public function testCanCreateFileLogger()
    {
        $loggerFactory = new FileLoggerFactory(sys_get_temp_dir());
        $logger = $loggerFactory->createLogger();
        $this->assertInstanceOf(FileLogger::class, $logger);
    }
}
```
